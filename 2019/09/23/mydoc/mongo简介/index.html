1.Mongo数据库介绍

1.1 NoSql

MongoDB属于NoSql数据库，NoSql数据库和传统的关系型数据库不一样，属于非关系型数据库。

举个简单的例子，假如我们有一个user表，用来存储用户的信息，其中包括uid、name、age，那么在MySQL这样的关系型数据库中，他可能长这样：

字段 类型
uid int
name varchar(50)
age int

在MongoDB中他可能长这样

看上去似乎没什么区别，如果我们有一个需求，需要在数据库中存储用户的自定义信息，让用户自己来添加，比如头像、地区、爱好之类的，那么MySql就有点头疼了，你可能需要先在数据库中添加三个字段，然后才能进行Inset操作。

在MongoDB里面，我们什么都不需要操作，只需要向数据库里面写数据就行了，数据的存储没有固定的模式，数据的横向扩展是很容易的。

1.2 连接

用工具，用代码，无论用啥，要使用数据库需要先进行连接，语法：

  • mongodb:// 这是固定的格式，必须要指定。
  • username:password@ 可选项，如果设置，在连接数据库服务器之后，驱动都会尝试登陆这个数据库
  • host1 必须的指定至少一个host, host1 是这个URI唯一要填写的。它指定了要连接服务器的地址。如果要连接复制集，请指定多个主机地址。
  • portX 可选的指定端口，如果不填，默认为27017
  • /database 如果指定username:password@，连接并验证登陆指定数据库。若不指定，默认打开 test 数据库。
  • ?options 是连接选项。如果不使用/database，则前面需要加上/。所有连接选项都是键值对name=value，键值对之间通过&或;（分号）隔开

这其实就是一个标准的URL格式。

1.5 创建集合

MongoDB数据库中，用集合(collection)来表示关系数据库中的表(table)，通常情况下

  • name: 要创建的集合名称
  • options: 可选参数, 指定有关内存大小及索引的选项

通常情况下，MongoDB不需要事先创建集合，系统会在用户插入文档的适合自动创建，当然，处于规范，一般会由DBA先创建需要的集合以及相应的索引

1.7.1 ObjectId

MongoDb会为每一条文档添加一个_id字段，类型是ObjectId，类似于MySql的自增字段。

MongoDB设计之初就是用于分布式环境，如果用自增字段，不同的数据库实例之间就会生成重复的id，为此，MongoDB采用了一个称之为ObjectId的类型来做主键。ObjectId是一个12字节的 BSON 类型字符串。按照字节顺序，依次代表：

4字节：UNIX时间戳
3字节：表示运行MongoDB的机器
2字节：表示生成此_id的进程
3字节：由一个随机数开始的计数器生成的值

1.8 更新文档

  • query : update的查询条件，类似sql update查询内where后面的。
  • update : update的对象和一些更新的操作符（如$,$inc…）等，也可以理解为sql update查询内set后面的
  • upsert : 可选，这个参数的意思是，如果不存在update的记录，是否插入objNew,true为插入，默认是false，不插入。
  • multi : 可选，mongodb 默认是false,只更新找到的第一条记录，如果这个参数为true,就把按条件查出来多条记录全部更新。
  • writeConcern :可选，抛出异常的级别。

1.9 删除文档

  • query :（可选）删除的文档的条件。
  • justOne : （可选）如果设为 true 或 1，则只删除一个文档，如果不设置该参数，或使用默认值 false，则删除所有匹配条件的文档。
  • writeConcern :（可选）抛出异常的级别。

1.10 查询文档

  • query ：可选，使用查询操作符指定查询条件
  • projection ：可选，使用投影操作符指定返回的键。查询时返回文档中所有键值， 只需省略该参数即可（默认省略）。
